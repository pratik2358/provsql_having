\section{Semantics}
\label{sec:semantics}
\pierre{1 page}

In the semimodule approach presented in~\cite{amsterdamer2011provenance}, provenance for aggregation is made algebraically uniform and compositional by lifting the aggregate to act on $\mathbb{K}$-annotated input tuples. 
This lifting is canonical only for aggregates with the right algebraic structure (specifically monoid aggregates, or aggregates reducible to monoid states such as \sql{AVG} via \sql{SUM} and \sql{COUNT}).
Thus, the semimodule approach of~\cite{amsterdamer2011provenance} for aggregate comparisons gives us a symbolic semimodule element for an aggregate for cases allowing this lifting. 
However, the semimodule approach fails for arbitrary operations: division, products of aggregates, conditional expressions, nested expressions e.g., \sql{AVG(x)+MEDIAN(y))/MAX(z)}.
There is no generic approach to apply these operations on semimodule expressions without imposing extra algebraic structures or adding ad hoc rules.

In this section we give a general \emph{possible world semantics} for selection predicates involving comparisons between aggregates.
This semantics does not need an algebraic lifting of the aggregate function.
Instead, for each possible subset of tuples, called a \emph{possible world}, in an aggregate group we evaluate the aggregate as an arbitrary function on the multiset of values and then restrict the world's provenance annotation to the satisfying cases only.
Thus any SQL aggregate function (average, median/percentile, array/JSON aggregates, custom UDFs, etc.) can be handled in this semantics.
Also, we lose explanations for  comparisons on aggregates, because atomic aggregate comparisons are treated itself as semimodule elements which are only evaluated after fixing the lifting homomorphism. \sil{I don't understand the previous sentence.}
Our semantics naturally support arbitrary arithmetic over arbitrary aggregate functions in \sql{HAVING}-like conditions; they are evaluated on the aggregate values in each possible world, and then the provenance is accumulated only for the tuples for which the comparison is true.

We fix $(\mathbb{K},\oplus,\otimes,\ominus,\delta,\mathbb{0},\mathbb{1})$ as our
annotation structure, as discussed in the preliminaries, along with a finite data domain $\mathcal{V}$.
For any set $S$, we write $\Bag(S)$ for the set of finite multisets over $S$.

First we introduce the notion of an arbitrary aggregate function:
\begin{definition}[Arbitrary Aggregate Function]
  An arbitrary aggregate function is a function  $f:\Bag(\mathcal{V})\to \mathcal{A}$, for some output domain $\mathcal{A}$.
\end{definition}
Note that monoid aggregate functions (\sql{SUM}, \sql{COUNT}, etc.) can also be treated as arbitrary aggregate functions in our semantics.
For comparing the result of two aggregates we define a comparison operator $\op$:
\begin{definition}[Comparison Operator]
  For any $\op\in\{=,\neq,<,\leq,>,\geq\}$ defined for a domain $\mathcal{A}$, we define
\[
  \chi_{\op}:\mathcal{A}\times\mathcal{A}\to \mathbb{K},
\qquad
  \chi_{\op}(a,b)\defeq
\begin{cases}
  \mathbb{1}\in \mathbb{K} & \text{if } a \op b,\\
  \mathbb{0} \in \mathbb{K} & \text{otherwise},
\end{cases}
\]
where $\mathbb{K}$ is any commutative $m-$semiring.
\end{definition}
Our goal is to add support to \sql{HAVING}-like queries in ProvSQL by extending the semiring-provenance semantics of \emph{selection} with possible-world semantics for \emph{selection}s whose predicates involve boolean combinations of comparisons between results of arbitrary aggregate functions.
We now introduce \emph{aggregate comparison atoms} in \emph{selection} conditions:
\begin{definition}[Aggregate Comparison Atoms]\label{def:agg_comp_atoms}
 Aggregate-comparison atoms are expressions of one of the forms
\[
(t:f) \op (t':f')
\;or\;
(t:f) \op c
\]
where $t,t'$ are terms of max-index $\le k$, $f,f'$ are aggregate functions with common output domain~$\mathcal{A}$,
$c\in\mathcal{A}$, and $\op\in\{=,\neq,<,\le,>,\ge\}$ and appear inside a
selection condition $\psi$ of queries.
\end{definition}

\begin{definition}[Possible-World]
Let $\hat I$ be a $\mathbb{K}$-instance. For $k\in \mathbb{N}$, a subquery $q\in\RA_k$, distinct grouping indices $(i_j)_{1\leq j \leq m}$ s.t. $1\leq i_j \leq k$ and any group key $\vec v=(v_1,\dots,v_m)\in\mathcal{V}^m$, define
the multiset of occurences of tuples involved as:
\[
U^{\hat I}_{q,\vec v} \defeq \mset{\, (u,\alpha)\in \angsem{\hat I}{q} \Big| (u_{i_1},\dots,u_{i_m})=\vec v \,}.
\]
A \emph{possible-world} for $\vec v$ is any $W\msubseteq U^{\hat I}_{q,\vec v}\, ,$ i.e, any $W$ over
$\mathcal V^k\times K$ such that $\forall(u,\alpha)$, $W(u,\alpha) \le U^{\hat I}_{q,\vec v}(u,\alpha)$.
\end{definition}

\begin{definition}[Possible World Semantics]
  Let $\hat I$ be a $\mathbb{K}$-instance. Fix $k\in \mathbb{N}$, a subquery $q\in\RA_k$, distinct grouping indices $(i_j)_{1\leq j \leq m}$ s.t. $1\leq i_j \leq k$ and any group key $\vec v\in\mathcal{V}^m$. For any \emph{possible-world} $W\msubseteq U^{\hat I}_{q,\vec v}$, we define it's $\mathbb{K}-$annotation as:
\[
\ann^{\hat I}_{q,\vec v}(W) \defeq
\Bigl( \bigotimes_{(u,\alpha)\in W}\alpha \Bigr)
 \otimes
\Bigl( \mathbb{1} \ominus \bigoplus_{(u,\alpha)\in U^{\hat I}_{q,\vec v}\msetminus W}\alpha \Bigr).
\]
(Empty $\otimes$-products equal $\mathbb{1}$; empty $\oplus$-sums equal $\mathbb{0}$.)
\end{definition}
\begin{definition}[Aggregate Bags]
Let $t$ be a term of max-index $\le k$ and $\hat I$ a $\mathbb{K}$-instance.
  For a possibble-world $W\msubseteq U^{\hat I}_{q,\vec v}$, define the \emph{induced multiset of values} $\in \Bag(\mathcal{V})$ as:
\[
\vals_t(W) \defeq \mset{\, t(u) \mid (u,\alpha)\in W\,}
\]
\end{definition}
\begin{definition}[Semantics of Selection Predicate]
Fix a $\mathbb K$-instance $\hat I$  and let $q\in \RA_k$.
A \emph{selection predicate} $\psi$ over arity $k$ induces a function
  $\predsem{\hat I}{q}{\psi} : \mathcal V^k \to \mathbb{K}$,
defined by recursion on~$\psi$ as follows:

\smallskip\noindent
For each atomic comparison, fix $\op\in\{=,\neq,<,\le,>,\ge\}$ with $R_{\op}\subseteq \mathcal V\times \mathcal V$, a
binary relation (possibly partial) on $\mathcal V$.
We assume that atomic comparisons are well-defined: whenever an atomic comparison between terms of
max-index $\leq k$,
  $t\op t'$ (or $t\op c \text{ where } c\in \mathcal V $) is evaluated on $u\in\mathcal V^k$, $t(u)$ and $t'(u)$ (or $c$) must lie in the domain on
which $\op$ is defined.
For an atomic comparison $\psi$ of the form $t\op t'$ (resp.\ $t\op c$ with $c\in\mathcal V$),
we write $\psi(u)$ for the truth value defined by
\[
\psi(u)\text{ holds}\Longleftrightarrow (t(u),t'(u))\in R_{\op}
\qquad
(\text{resp. }(t(u),c)\in R_{\op}).
\]
If $\psi$ is of such an atomic form:
\[
\predsem{\hat I}{q}{\psi}(u)\defeq
\begin{cases}
\mathbb 1 & \text{if }\psi(u)\text{ holds,}\\
\mathbb 0 & \text{otherwise.}
\end{cases}
\]
\smallskip\noindent
If $q$ is of the form $\gamma_{i_1,\dots,i_m}[t_1:f_1,\dots,t_n:f_n](q')\in \RA_{k=m+n}$
with $q'\in\RA_{k'}$ and distinct grouping indices $(i_j)_{1\le j\le m}$ such that $1\le i_j\le k'$, 
consider aggregate-comparison atom(s): $(t:f)\ \op\ (t':f')$ or $(t:f)\ \op\ c$ with $c\in\mathcal V$ and max-index of $t,t'$ being $\leq k'$.

\smallskip\noindent
For $u=(\vec v,a_1,\dots,a_n)\in\mathcal V^{m+n}$ ,
write $\vec v=(v_1,\dots,v_m)$ for the group key.

\smallskip\noindent
For $\psi = (t:f) \op (t':f')$,
\[
\predsem{\hat I}{q}{\psi}(u)\defeq
  \bigoplus_{\substack{W\msubseteq U^{\hat I}_{q',\vec v}\\ W\neq\emptyset}}
\Bigl(
\ann^{\hat I}_{q',\vec v}(W)\otimes
\chi_{\op}\bigl(f(\vals_t(W)),\ f'(\vals_{t'}(W))\bigr)
\Bigr).
\]

\smallskip\noindent
For $\psi = (t:f) \op c$,
\[
\predsem{\hat I}{q}{\psi}(u)\defeq
  \bigoplus_{\substack{W\msubseteq U^{\hat I}_{q',\vec v}\\ W\neq\emptyset}}
\Bigl(
\ann^{\hat I}_{q',\vec v}(W)\otimes
\chi_{\op}\bigl(f(\vals_t(W)),\ c\bigr)
\Bigr).
\]

\smallskip\noindent
Now, for $u\in\mathcal V^k$, let $\psi_1$ and $\psi_2$ be arbitrary selection predicates. Then:
\[
\predsem{\hat I}{q}{\psi_1\land \psi_2}(u)\defeq
\predsem{\hat I}{q}{\psi_1}(u)\otimes \predsem{\hat I}{q}{\psi_2}(u),
\]
\[
\predsem{\hat I}{q}{\psi_1\lor \psi_2}(u)\defeq
\predsem{\hat I}{q}{\psi_1}(u)\oplus \predsem{\hat I}{q}{\psi_2}(u).
\]
\[
\predsem{\hat I}{q}{\neg \psi}(u)\defeq \mathbb 1 \ominus \predsem{\hat I}{q}{\psi}(u).
\]

\end{definition}


\begin{definition}[Provenance for HAVING-like queries]Let $\hat I$ be a $\mathbb K$-instance, let $k\in\NN$, let $q\in\RA_k$, and let $\psi$ be
a well-formed selection predicate over arity $k$.
We define:
% \[
% \angsem{\hat I}{\sigma_\psi(q)} \defeq
% \mset{\, (u,\alpha\otimes \predsem{\hat I}{q}{\psi}(u)) \mid (u,\alpha)\in \angsem{\hat I}{q}\,}.
% \]
%
\[
\angsem{\hat I}{\sigma_\psi(q)} \defeq
\begin{cases}
\mset{\, (u,\alpha\otimes \predsem{\hat I}{q}{\psi}(u)) \mid (u,\alpha)\in \angsem{\hat I}{q}\,}
& \text{if $\psi$ contains no aggregate-comparison atom,}\\[1ex]
\mset{\, (u,\predsem{\hat I}{q}{\psi}(u)) \mid (u,\alpha)\in \angsem{\hat I}{q}\,}
& \text{otherwise.}
\end{cases}
\]
\end{definition}

\smallskip\noindent\emph{Note:}
For any aggregate comparison atom, if $U^{\hat I}_{q',\vec v}=\emptyset$ then every output
tuple $u$ with group key $\vec v$, we have $\predsem{\hat I}{q}{\theta}(u)=\mathbb 0$.
Since the possible-world semantics
ranges over non-empty worlds only.
So multiplying the possible world provenance
expression with $\alpha$( which is a $\delta(\cdot)$-type provenance expression to enforce group existence) is redundant.
